# Transformer / LLM 原理筆記

> 從「LLM 是什麼架構」一路推到親手算 attention 的完整脈絡。
> 學習方式：先抓直覺 → 具體小數字手算 → 看 shape 對不對。

---

## 0. 三個最容易混的詞（先釘死）

| 名詞 | 它是什麼 | 類比 |
|------|---------|------|
| **Transformer** | 架構藍圖（零件怎麼接） | 房子的**設計圖** |
| **GPT** | 一種「只用 decoder」的 Transformer + 訓練好的數字 | 蓋好裝潢完的**房子** |
| **訓練** | 把藍圖裡的數字從亂數調到好用 | **裝潢的過程** |

- **架構 = 人寫死的，永遠不變**（決定數字放在哪些格子）。
- **參數 = 一坨數字**（決定格子裡填什麼）。訓練只動參數。
- 訓練好的模型 = 一個幾 GB 的檔案（`.safetensors`），裡面就是那坨數字。
- Llama-7B 的「7B」= 70 億個這種數字。

---

## 1. 架構：現代 LLM 是 Decoder-only

| 架構 | encoder | decoder | 代表 |
|------|---------|---------|------|
| 原始 Transformer (2017, 翻譯) | ✅ | ✅ | 翻譯模型 |
| **Decoder-only**（主流 LLM） | ❌ | ✅ | GPT、Llama、QLoRA 練的那種 |
| Encoder-only | ✅ | ❌ | BERT（分類/embedding） |

> ⚠️ GPT **沒有 encoder**。它是 N 個一模一樣的 Block 疊起來（Llama-7B 是 32 層），不是「無數個 encoder/decoder」。

每個 Block 內部只有兩個主零件：
```
   ┌──────────── Transformer Block ────────────┐
   │   x ──► [ Self-Attention ] ──► (+x) ──►    │
   │              ──► [ MLP/FFN ] ──► (+) ──► out│
   └────────────────────────────────────────────┘
```

## 2. 完整 Pipeline（背這張）

```
"貓 熊"
   │  ① tokenizer：切成 id [貓id, 熊id]
   ▼
查 embedding 表：id → 向量      shape (2, n)   ← 一進（查表＝用，不改）
   │
   ▼  ② 核心理解
[ attention + FFN ] × 32 層      ← 每層讓字互相吸收上下文
   │
   ▼
拿「最後一個字」的向量            shape (n,)    ← 只用最後一個位置去預測下一字
   │
   ▼  ③ 出口
unembedding 投影 → logits (約5萬個分數)   ← 一出（向量→全字典分數）
   │
   ▼  softmax → 機率 → top_k 取樣
選出下一個字「跑」
   │
   ▼  ④ 接回句尾 "貓熊跑"，整段重跑一次（自回歸 autoregressive）
循環，直到產生結束符
```

四步：**切字 → 查表進場 → 32層理解 → 投影出場挑字 → 接回去再來**。

## 3. Embedding（入口查表）

- **是什麼**：一張查表（參數矩陣），`token id → 向量`。**動作很笨**（純查字典），但**內容很珍貴**。
- **它「encode」字（動作），但不是、也不需要叫「Encoder」的元件。** 別跟 day2 RAG 的 embedding 模型搞混（那是另一個獨立模型，常是 encoder-only）。
- **怎麼來**：
  - 初始值 = **亂數**（小小的隨機數）。此時「貓」「狗」向量八竿子打不著。
  - 跟整台 GPT 一起，用「猜下一個字」訓練。**前向查表用它 → 預測 → 算誤差 → 誤差倒流(backprop)校正它**，重複幾兆次。
  - 訓練完**凍結**，之後只查不改（QLoRA 是特例：繼續微調，所以又會動一點）。
- **沒有通用基準表**：每個模型自己從亂數練自己的。（歷史上 word2vec/GloVe 有過通用詞向量，現代 GPT 不這樣。）

### 數字的意義在「幾何位置」，不在單一數字
- 單一維度（某個 0.2）通常沒有人類看得懂的意義。
- 意義藏在**相對位置**：
  ```
  距離 = 語意相似度：  貓 ● ● 小貓  ● 狗 ........ ● 股票
  方向 = 語意關係：    國王 - 男人 + 女人 ≈ 皇后
  ```
- 機制叫**分佈假說**：「一個字的意思由它常跟哪些字一起出現決定。」出現在相似上下文 → 向量被推到相似位置（猜下一個字的副產品）。

### Embedding 是「無上下文」的
- token embedding 只看字本身，看不到旁邊 → 「貓熊的貓」和「我家的貓」在這階段向量**一模一樣**。
- 「貓熊 = 熊」這種上下文理解，是**後面 attention 層算出來的**，不是 embedding。

> 重點：embedding = 入口（給無上下文起點）；智能/理解在後面 32 層 attention+FFN。

## 4. Attention（最精華：把上下文「算」進來）

### 直覺（搜尋引擎比喻）
```
Q（query）= 我打進去的「搜尋關鍵字」  →「我想找什麼」
K（key）  = 每個 token 掛的「標籤」    →「我能提供什麼」（拿來跟 Q 比對）
V（value）= 每個 token 真正的「內容」  →「選我的話給你的料」
```
- **V 是被混合的「原料」，不是結果。** Q·K 決定「聽誰、聽多少」(權重)，再把大家的 V 按權重混合，混出來才是 output。

### 三步驟
```
1. scores = Q · 每個 K（內積）         → 量「方向多像」= 相關性分數
2. weights = softmax(scores)           → 變成加總=1 的權重
3. output  = weights 加權混合 V         → 吸收了上下文的新向量
```

### 為什麼內積能當相關性分數
> 兩向量方向越像 → 內積越大；越不相干 → 越小（甚至負）。
```
貓[1,0]·熊[0.8,0.6] = 0.8（高）→ 熊跟我相關
貓[1,0]·跑[0,1]     = 0.0（低）→ 跑跟我無關
```
（真實公式 scores 還要 ÷√d，避免維度大時分數爆炸害 softmax 變極端；手算範例為求乾淨省略。）

### 其他名詞
- **Multi-Head (MHA)**：同時做好幾組 QKV（如 32 head），各學不同關係，再拼起來。
- **Causal Mask（因果遮罩）**：decoder 做「猜下一字」，第 3 字不准看到第 4、5 字（否則作弊）→ 未來位置分數設 -∞。
- **現在的優化**：GQA（共享 K/V 省記憶體）、FlashAttention（算更快）、RoPE（位置編碼）。

## 5. Softmax = exp 後看比例

```
scores  = [1.0,  0.8,  0.0 ]
取 exp  → [2.72, 2.23, 1.0 ]      (e^1.0, e^0.8, e^0.0)
加總    = 5.95
比例    → [0.457, 0.374, 0.168]   (每個 ÷ 5.95，加起來=1)
```
**為什麼要先 exp？**
1. **保證正數**：內積可能負，直接算比例會出現「負機率」💥；exp 把全部變正。
2. **放大領先者**：exp 是指數，分數高一點 exp 後高很多 → attention 能「該專注時就專注」，不雨露均霑。

## 6. FFN（每層的另一半）

- 每個 token 各自做的非線性運算。
- 研究發現它像**知識庫**：「巴黎→法國首都」這種事實，存在 FFN 的參數裡。
- 參數量大頭在 attention + FFN 這 32 層，token embedding 只占一小部分。

## 7. 輸出端：unembedding → 取樣

```
最後向量 → unembedding（反向投影）→ logits: 字典每個字一個分數（約5萬個）
        → softmax → 機率
        → 取樣策略選出 1 個字
```
- **輸入端 vs 輸出端**：輸入是「id→向量」(查)；輸出是「向量→全字典分數」(投影)。方向相反。
- **weight tying**：很多模型讓輸入查表和輸出投影**共用同一張表**（一次正用、一次反用）。所以「輸出像在查 embedding」的直覺半對。
- **取樣參數**：
  - `top_k`：只留機率最高的前 k 個，其餘歸零再抽。
  - `top_p`：留累積機率到 p 的那群。
  - `temperature`：調分佈尖銳/平緩。

## 8. 訓練：怎麼從亂數變會說人話

### 核心任務：猜下一個字（next-token prediction，自監督）
- 遮住下一個字叫模型猜。**正確答案＝真實文章的下一個字，免費附在資料裡**，不需人工標註 → 才能餵幾兆字。
- 配合 causal mask，一句話能同時造出 N 個「猜下一字」練習題。

### 「一開始全是亂數，怎麼校正？」
> 校正目標不是「一張正確的表」（沒有），而是「把真實文章的下一個字猜對」。
```
模型(亂數)猜「子」機率 0.00002 → loss 很大
→ 梯度(gradient)：每個數字「調大一點，loss 會升還降？」→ 往降的方向踏一小步
→ 重複幾兆次
```
- **矇眼下山比喻**：loss = 山高（越低越好），亂數 = 隨機丟在山上。看不到山谷，但能用腳感覺坡度(梯度)，往下坡踏一小步。走幾兆步，從亂數走到會說人話的低點。**一開始在哪不重要，一直往下走就好。**

### 雜訊 / 資料品質
- 偶爾一句鬼話：因「每次只走一小步 × 正常資料量碾壓」→ 被平均淹沒，沒事。
- 模型學的是**機率分佈**而非唯一答案（「貓坐在」後面可接墊子/地上/沙發…）→ 多種合理續寫看似矛盾，其實正是它要學的。
- 但垃圾資料若**佔多數** → 學成垃圾（garbage in, garbage out）→ 所以**洗資料(data curation)是 LLM 命脈**，面試可講。

### 官方三階段訓練
```
① Pre-training：餵幾兆字純猜下一字 → Base Model（知識多但只會接話）★最貴
② SFT（監督微調）：餵人寫的問答對 → 教它「回答問題」      ← QLoRA 在這
③ RLHF / DPO：人類標「哪個答案好」 → 對齊人類偏好＋安全
```

## 9. 數學地基（看 shape 比手算重要）

> AI Engineer 95% 時間在「看矩陣形狀對不對」，不是腦中手算內積。numpy 幫你算。

### Shape 怎麼讀
- 形狀寫成 `(幾個橫排 row, 每排幾個 column)`。
- **數最外層括號層數 = 幾維**：一層=向量、兩層=矩陣。
```
[1, 2, 3]        shape (3,)     1維：一條線，無「排」概念
[[1, 2, 3]]      shape (1, 3)   2維：1橫排 × 3個
[[1],[2],[3]]    shape (3, 1)   2維：3橫排 × 1個
```
- 實戰讀法：`(3, 2)` = 「3 個字，每字 2 維」= (序列長度, 維度)。
- 報 `shape mismatch` 時第一招：`print(x.shape)` 數逗號。

### 三個小工具
- **內積 dot**：對齊位置相乘再全加。`[1,0]·[0.8,0.6] = 1×0.8+0×0.6 = 0.8`。`np.dot()`。
- **單位矩陣 identity**：對角線1其餘0，「乘了等於沒乘」（數字界的1）。手算範例把 W 設成它＝「這步先跳過」。
- **exp**：`e≈2.718` 的次方，`np.exp()` 一個按鈕。作用：全變正＋大的拉更大。永遠不用手算。

### 向量四種運算的物理意義
| 運算 | 吃什麼 | 吐什麼 | 物理意義 | AI 裡 |
|------|--------|--------|----------|-------|
| **內積** dot | 兩向量 | 一個**數字** | 同方向程度 / 投影 | 相似度、attention 分數 |
| **叉積** cross | 兩個3D向量 | 一個**垂直向量** | 旋轉/力矩/面積 | 幾乎不用 |
| **外積** outer | 兩向量 | 一個**矩陣** | 所有配對攤平(rank-1) | **LoRA** |
| **矩陣乘法** | 矩陣×向量 | 向量/矩陣 | **變換**(旋轉拉伸投影) | 每層權重在變換向量 |

- **內積**：`a·b = |a||b|cos θ`。收縮成一個數，量「方向多像」。
- **外積(outer)**：直向量⊗橫向量→矩陣（所有配對相乘攤平），是 rank-1。**LoRA 的 `ΔW ≈ B@A` 就是用少數外積近似巨大權重變化** → 所以「只調一點點參數就有效」。
- **叉積(cross)**：3D 物理用（力矩/法向量），AI 用不到。注意中文「外積」歧義：ML 講的是 outer，不是 cross。
- **矩陣乘法**兩個看法：
  - (A) 算法：`C[i,j] = A第i橫排 · B第j直行`（一堆內積）→ 解釋 shape 規則 `(m,k)@(k,n)=(m,n)`。
  - (B) 物理意義 ⭐：**矩陣 = 一台把向量旋轉/拉伸/投影的變換機器**。`q = x @ W_q` 就是「W_q 把 x 變換到查詢空間」。神經網路的「思考」= 向量被一個個學好的矩陣反覆扭，最後扭到能預測下一字。（手算範例把 W 設成單位矩陣＝不做變換，所以 q=k=v=x。）

### cos 只要記一句
- 三角形定義：`cos θ = 鄰邊/斜邊`（SOH-CAH-TOA）。
- **更好用的單位圓直覺**：長度1的箭頭轉 θ 角，`cos θ = 尖端的 x 座標`。
  ```
  θ=0°→cos=1（完全同向）  θ=90°→cos=0（垂直/無關）  θ=180°→cos=-1（反向）
  ```
- **要記的就一句：cos = 同方向程度，範圍 -1~1。** 數值永遠用 `np.cos()`，不用背。
- 這就是為何內積能當相關性：`a·b` 裡含了 cosθ，從 1→-1 把「超像→無關→相反」描完。

## 10. 手算 Attention 範例（親手跑一次最有感）

3 個 token、2 維、W_q=W_k=W_v=單位矩陣（所以 q=k=v=x），只算「貓」的輸出：

```python
import numpy as np

x_cat  = np.array([1.0, 0.0])   # 貓
x_bear = np.array([0.8, 0.6])   # 熊（跟貓有點像但不同）
x_run  = np.array([0.0, 1.0])   # 跑（離貓很遠）
V = np.stack([x_cat, x_bear, x_run])   # (3,2)；因 W=I，q=k=v=x

# Step1 scores = 貓的 q 對每個 k 內積
scores = np.array([1.0, 0.8, 0.0])     # [貓·貓, 貓·熊, 貓·跑]

# Step2 softmax
weights = np.exp(scores) / np.sum(np.exp(scores))
# → [0.457, 0.374, 0.168]   熊(0.374) > 跑(0.168) ✅ 符合相關性

# Step3 加權混合 V
output_cat = weights @ V                # (3,) @ (3,2) → (2,)
# → [0.757, 0.393]
```

**結果解讀**：
```
進去：貓 = [1, 0]        ← 無上下文的純「貓」
出來：貓 = [0.757, 0.393] ← 第二維從 0 長出 0.393，被往「熊」拉過去
```
> 這就是「貓熊的貓知道自己旁邊有熊」的瞬間。一層一個 head 做一次，疊 32 層 × 多 head 做幾百次 = Transformer 的「理解」。

weights 對應靠**位置(index)**：`weights[0]=貓對貓, [1]=貓對熊, [2]=貓對跑`（陣列本身不帶標籤，要自己記 token 順序）。
