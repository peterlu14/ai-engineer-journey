# 製造業測試指標：良率 / mean / sigma / Cp / Cpk

> 這是面試的差異化武器。一般 AI 轉職仔講不出來，但我有 CP test 場域，講得出「用統計指標定位產線問題」就贏。
> 例子都用 demo 的 spec：VF（LSL=1.00, USL=1.45, μ≈1.22, σ≈0.06）、CTR（LSL=80, USL=180, μ≈130, σ≈18）。

---

## 1. 良率 Yield

```
yield = pass 數 / total 數
```

**坑：lot 良率 ≠ die 良率，也 ≠ 單項 pass 率。**

- 一顆 die 要「**每一項 test item 都 pass**」才算 pass。
- VF pass 但 IR fail → 這顆 die 是 **fail**。
- 所以不能直接對 `pass_fail` 欄位平均，要先 `group by die`，再看「這顆 die 是不是每項都過」。

**Debug 線索：** 若各項獨立，
```
die 良率 ≈ VF率 × IR率 × CTR率 × BV率
```
若某 lot 的 VF 單項 = 99%，但 die 良率只有 92% → **不是 VF 害的**，是其他項在「VF 有過的那些 die」上把它們刷掉。
→ 結論：別調 VF，去查**良率最低的那一項**。

---

## 2. mean / sigma —— 為什麼看 σ 不只看「有沒有過」

- **良率是落後指標**：只告訴你「現在過了幾顆」。
- **σ 是領先指標**：告訴你「製程穩不穩、會不會明天開始 fail」。

VF：σ=0.06 → 大部分落在 μ ± 3σ = `1.22 ± 0.18` = `1.04 ~ 1.40`，還在 spec 內。
若哪天 σ 變 0.12 → ±3σ = `0.86 ~ 1.58`，**已超出 `1.00~1.45`**。
→ 良率還沒掉，但已經在漏了。**這就是為什麼良率 99% 還要盯 σ。**

---

## 3. Cp vs Cpk —— 製程能力指數（半導體必考）

把 σ 跟 spec 寬度綁成一個數字。

### Cp（製程「潛力」，不管偏不偏）
```
Cp = (USL - LSL) / 6σ
```
意思：spec 寬度能塞下幾個 6σ。Cp 越大越好。
**盲點：假設你正好在中心。**

VF：`(1.45 - 1.00) / (6 × 0.06) = 0.45 / 0.36 = 1.25`

### Cpk（製程「能力」，有考慮中心偏移）
```
Cpk = min( (USL - μ) / 3σ , (μ - LSL) / 3σ )
```
取「離較近那一邊 spec」的餘裕（較差的那邊）。

VF：`min( (1.45-1.22)/0.18 , (1.22-1.00)/0.18 ) = min(1.28, 1.22) = 1.22`
→ 中心 1.22 偏向 USL，所以 Cpk(1.22) < Cp(1.25)。

CTR：μ=130 剛好在 `80~180` 正中央 → Cp = Cpk = `100/108 = 0.93`（沒偏移，兩者相等）。

### 核心一句（背起來）
> **Cp 是「製程夠不夠精」，Cpk 是「製程又精又準」。**
> **Cp 高但 Cpk 低 = 機台很穩但中心偏了 → 該調機，不是換機。**

### 業界門檻
| Cpk | 意義 |
|-----|------|
| ≥ 1.67 | 優秀 |
| ≥ 1.33 | OK |
| < 1.00 | **正在量產不良品**（連 ±3σ 都包不住，>0.27% 落在規格外）|

→ CTR 的 Cpk=0.93 < 1.0：該示警。救法是縮 σ（製程改善）或放寬 spec（跟客戶談），不是調機就能解。

**重點：Cpk 數字要會「翻譯成行動」，光算出來不夠。**

---

## 4. Wafer map —— 不是統計，是「空間」

前三個看「數字」，wafer map 回答：**fail 是隨機的，還是集中在某位置？**

- 隨機散 → 多半是量測雜訊。
- 集中邊緣 / 某一角 → 製程問題（鍍膜不均、邊緣效應）。

前端 heatmap 要的資料：對每個 `(x, y)` 算「fail 了沒 / fail 率」。
→ 這就是 schema 要存 `x, y` 的原因。
